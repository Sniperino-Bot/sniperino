Rules

Flexible Rule Engine

Each Telegram source gets its own configurable rule. Set buy amount, cooldown, auto-sell multiplier, stop-loss percentage, and trailing mode independently per channel or user. Three matching modes: Private DM, user inside a group, or any post in a group.

Reliability

Tri-Source Price Monitoring

Active positions polled every 2 seconds via Jupiter Price API v3, DexScreener fallback, and Helius on-chain data. A single batched call covers all open positions. If one source is rate-limited or down, the next takes over — auto-sell triggers stay alive.

Edge

Pre-Migration Detection

Sniperino computes the bonding curve PDA for Pump.fun tokens and monitors it via WebSocket. Raydium and PumpSwap migration events are detected before they fully finalize on-chain — giving you an earlier entry window than any post-migration sniper.

Control

Live Trade Modifications

Auto-sell targets, stop-loss prices, and trailing stop-loss can be updated from the GUI while trades are live. Changes take effect on the next price tick. No restarting, no lost positions, no missed exits. Partial sells at 25%, 50%, 75%, or 100% from a single popup.

Zero-Delay SOL Price Warmup
A dedicated daemon pre-fetches the SOL/USD price at startup and refreshes every 30 seconds. Buy orders use a cached, always-ready price — no blocking lookup at trade time.
Batched Position Polling
All open positions are covered with a single Jupiter API call per 2-second cycle. API rate-limit exposure stays flat regardless of how many concurrent trades are active.
Retry-Until-Success Logic
Buys retry up to 10 times (backoff capped at 12s). Sells retry up to 5 times with 80-second Helius confirmation. 429 and 5xx errors handled gracefully — the trade opportunity is never abandoned on a transient failure.
Tunable Priority & Slippage
PRIORITY_FEE_LAMPORTS (default 150,000) and DEFAULT_SLIPPAGE_BPS (default 100 bps) are configurable per deployment without touching code.
Multi-Trade Parallelism
MAX_PARALLEL_TRADES controls simultaneous buy/sell flows. Multiple signals from different channels execute concurrently without resource contention.
Security

Protected at
Every Layer.

From license validation before startup to atomic sell guards during execution — Sniperino is hardened throughout. Your wallet credentials and trade logic are protected by design.

HWID-Bound License
Every license key is tied to a unique hardware fingerprint (MD5 of the network adapter MAC). A key cannot run on more than one machine simultaneously — no unauthorized sharing.
Online Validation at Startup
Before the GUI loads or any API key is read, the license is validated against a remote server with up to 3 retries. Invalid, expired, banned, or HWID-mismatched keys are rejected — the bot will not start.
Periodic 24-Hour Re-Check
A background daemon re-validates the license every 24 hours while the bot runs. A countdown warning lets you enter a new key without restarting; after expiry the bot shuts down gracefully.
Atomic Sell Guards
Auto-sell and stop-loss triggers use lock-protected check-and-set operations. Each event is sold exactly once — even under concurrent price polling. No double-sells, no missed exits.
ENV Whitelist & File Hardening
The .env loader rejects any key not in a hardcoded whitelist — injected variables cannot reach the runtime. On Windows, icacls restricts .env access to the current user only (R/W), protecting wallet credentials from other OS users.
Trailing Stop-Loss & Liquidity Filter
Stop-loss floor rises dynamically as price sets new highs. Configurable liquidity threshold (default $6,500) filters out illiquid and honeypot tokens before a buy is ever attempted.
Forward Trust Policy & Duplicate Protection
Forwarded messages are attributed to the sender, not the original author — preventing spoofed forwards from impersonating trusted signal sources. Each mint is only bought once per session via an already_traded_mints guard.
